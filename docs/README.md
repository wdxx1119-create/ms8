- 可复现缺陷、功能请求和文档问题：使用仓库 Issue 模板。
- 安全漏洞：不要公开提交 Issue，请通过 GitHub Security Advisory 私密报告。
- 代码变更：创建 Pull Request，并完成仓库 PR 模板中的测试、安全和兼容性检查。
