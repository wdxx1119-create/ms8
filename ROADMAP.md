# MS8 Roadmap

MS8 当前处于 **Alpha** 阶段。本路线图用于说明优先方向，不构成固定发布日期承诺。实际顺序会根据安全问题、数据完整性、用户反馈和维护资源调整。

## 当前原则

1. 本地优先：核心记忆默认保存在用户控制的本地目录。
2. 治理优先：写入、读取、批处理和自动化不得绕过授权、审查和风险边界。
3. 可恢复：备份、迁移、校验和回滚能力优先于不可逆自动化。
4. 可验证：发布包、CLI、MCP、Absorb 和数据路径必须有自动化验证。
5. 渐进增强：外部或本地模型是可选增强，基础能力应有明确降级路径。

## Alpha 阶段重点

### 1. 数据完整性和恢复

- 稳定 JSONL、SQLite、知识图谱和索引之间的一致性检查
- 完善损坏记录隔离和可审计修复流程
- 增加备份恢复和跨设备迁移测试
- 明确数据格式兼容策略和迁移机制
- 为高风险重置、删除和回滚操作保留 dry-run

### 2. MCP 兼容性

- 持续验证官方支持的 MCP 客户端
- 增强配置生成、验证、smoke 和 rollback 流程
- 保证打包产物包含配置、adapter registry 和必要资源
- 明确客户端权限、错误处理和降级行为
- 增加协议兼容性和错误输入测试

### 3. Absorb 安全边界

- 强化授权目录、排除规则和高风险根目录确认
- 完善文件解析失败、OCR 缺失和恶意文件处理
- 提升待审、隔离、来源标记和按来源回滚能力
- 增加提示注入、敏感信息和超大文件相关测试
- 保持自动写入默认关闭

### 4. 跨平台稳定性

- 持续验证 Python 3.10–3.13
- 增强 Windows 路径、权限和虚拟环境测试
- 保持 macOS 和 Linux 隔离安装验证
- 处理包含空格、Unicode 和长路径的安装场景
- 减少系统级可选依赖造成的不透明失败

### 5. 可观测性和自维护

- 改善 `doctor`、dashboard 和健康报告的可操作性
- 区分故障、功能降级和可解释告警
- 为自修复策略补充预览、审计和回滚
- 避免日志泄漏记忆、密钥、路径和 PII
- 建立更清晰的稳定能力与 Labs 能力边界

### 6. 开发者体验

- 保持 Ruff、mypy、pytest 和打包验证全绿
- 建立覆盖率基线，再逐步引入合理门槛
- 完善 CONTRIBUTING、SUPPORT 和安全文档
- 增加架构、数据模型和威胁模型文档
- 提供可复制的最小示例和问题复现模板

## Beta 准入条件

项目进入 Beta 前，至少应满足：

- 当前支持平台不存在已知的数据丢失阻塞问题
- 备份、恢复和迁移流程有自动化测试与用户文档
- 核心 CLI、MCP 和 Absorb 路径具有稳定的回归测试
- 发布包在 Python 3.10–3.13 上通过构建和安装验证
- macOS、Linux 和 Windows 的关键流程有持续验证
- 安全报告和修复流程可用
- 数据格式和兼容性策略明确
- 核心模块建立覆盖率基线并持续改善
- Alpha 期间发现的高严重度问题已关闭或有明确缓解措施
- 用户可通过 Quick Start、FAQ、Use Cases 和 Support 文档独立完成常见操作

## Beta 之后

Beta 阶段预计重点关注：

- API 和配置稳定性
- 更明确的弃用和迁移周期
- 性能与大规模本地资料索引
- 更多 MCP 客户端兼容性
- 文档国际化
- 更成熟的发布和供应链安全流程

## 暂不承诺

以下方向可能探索，但不属于当前 Alpha 核心承诺：

- 托管式记忆云
- 绕过本地治理的自动同步
- 对所有第三方模型和客户端提供兼容保证
- 无人工确认的高风险批量修改
- 固定 SLA 或商业支持响应时间

## 提议路线图变更

路线图建议应说明：

- 用户问题和影响范围
- 为什么符合 local-first 与治理原则
- 所需数据迁移或兼容性成本
- 安全和隐私影响
- 可验证的完成标准

可通过 Feature Request Issue 提交明确建议；开放式方案讨论适合放在 Discussions（启用后）。
