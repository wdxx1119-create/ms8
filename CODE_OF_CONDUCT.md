# Code of Conduct

## 我们的承诺

MS8 社区致力于提供开放、友善、尊重且免受骚扰的协作环境，不因年龄、身体特征、残障、族群、性别认同与表达、经验水平、教育背景、国籍、外貌、种族、宗教、性取向或其他个人特征而区别对待。

参与者应共同维护一个以技术事实、建设性反馈、隐私保护和负责任披露为基础的社区。

## 期望行为

可接受的行为包括：

- 尊重不同经验、观点和使用场景
- 对代码和设计提出具体、可验证、面向改进的反馈
- 承认错误并主动修正
- 将讨论集中在项目、用户影响和技术权衡上
- 保护他人的个人信息、密钥、日志、记忆数据和未公开漏洞
- 在可能影响安全或隐私时优先使用私密报告渠道

不可接受的行为包括：

- 侮辱、威胁、歧视、骚扰或人身攻击
- 恶意打断讨论、持续挑衅或制造敌意
- 未经允许公开他人的身份信息、通信内容或私人数据
- 在公开 Issue、PR、Discussion 或日志中发布密钥、真实个人记忆、PII 或漏洞利用细节
- 利用项目渠道推广与项目无关的商业、政治或欺诈内容
- 任何在专业协作环境中明显不适当的行为

## 执行范围

本规范适用于仓库中的 Issue、Pull Request、Discussion、代码审查、发布说明和其他官方项目空间。代表项目参加外部活动或公开交流时同样适用。

## 报告与处理

普通行为问题可通过仓库维护者的私密联系方式报告。涉及安全漏洞、密钥泄漏、权限绕过或敏感数据暴露时，请使用 [SECURITY.md](SECURITY.md) 中的 GitHub Security Advisory 渠道，不要公开提交。

报告应尽量包含：

- 事件发生的位置和时间
- 相关内容或链接
- 对参与者或项目的影响
- 希望采取的处理方式

维护者会尽量保护报告者隐私，并根据严重程度采取提醒、内容编辑、临时限制、永久移除或向平台报告等措施。

## 执行责任

维护者有责任解释和执行本规范，并可删除、编辑或拒绝不符合规范的评论、提交、Issue、PR 或其他贡献。执行决定应基于可验证行为，而不是技术观点分歧本身。

## 归属

本规范参考 Contributor Covenant 2.1 的原则，并针对 MS8 的本地数据、隐私和安全披露场景进行了调整。
